The Scenario

There are instances when a task that initially took seconds to process becomes more time- consuming. Taking a manual image search for example, when there are only 10 images, it is quite easy to search. However, after accumulation of thousands of images, this manual task becomes overwhleming, especially when the image names may not be intuitive.

One solution for this is to use object detection algorithms to identify features within the images and use the resultant labels to query for a potential match. AWS Rekognition was used to detect the objects in the images and label each with a level of confidence.

The Goal

The goal was to leverage that manual process into something more automatic, where the communal images could be labelled by the object detection algorithm and have those labels persist to be queried from a suitable interface.

Lessons Learnt

Limited labels to represent an object

For example, ‘zebra-crossing’ was the only label returned to represent a pedestrian crosswalk. So, unfortunately if a user searched for ‘crosswalk’, no images would be returned.

Lack of Canadian context at times

Some labels seemed to be more suited for a European audience such as ‘tram’ and ‘trolley’. However, the term ‘light rail’ returns unrelated images, containing traffic lights or railings. Luckily, the label ‘train’ was available.

Keep Improving!

This is something we need to investigate further. Hopefully, AWS Rekognition’s custom labeling will help to alleviate some of these quirks in future iterations of this prototype.
